# 03 — Юридические страницы (152-ФЗ)

**Приоритет:** P0  
**Оценка:** 1 день  
**Зависимости:** [01-fix-navigation-and-routes](./01-fix-navigation-and-routes.md)

## Описание

Добавить обязательные для медицинского тендера юридические страницы. Сейчас в формах есть чекбокс согласия на обработку ПДн, но ссылка `/personal-conversations` ведёт на 404.

## Ограничение

Чекбоксы в `RegistrationForm` и `OrderConformationForm` **остаются** — меняется только URL ссылки и добавляется целевая страница. Логика валидации `acceptTermAndConditions` не меняется.

## Страницы

| URL | Содержание |
|-----|------------|
| `/personal-data-policy` | Политика обработки персональных данных (152-ФЗ): цели, категории данных, сроки хранения, права субъекта, контакты оператора |
| `/personal-data-consent` | Текст согласия на обработку ПДн (шаблон для ознакомления) |
| `/terms` | Пользовательское соглашение |
| `/license` | Лицензии и разрешительная документация: реквизиты юрлица, номера лицензий, ссылки/превью документов |

## Дополнительно

- Cookie-баннер (минимальный): уведомление об использовании cookies (`react-use-cookie`, `react-cookie` уже в проекте)
- Блок «Оператор персональных данных» в футере: наименование, ИНН, адрес, email DPO

## Контент

На этапе demo — шаблонные тексты с пометкой «Demo Clinic» и placeholder-реквизитами. Перед production — заменить юристом.

## Критерии приёмки

- [ ] Ссылка из формы регистрации открывает `/personal-data-policy`
- [ ] Ссылка из формы оформления заказа анализов открывает ту же или consent-страницу
- [ ] `/license` доступна из футера «Документы»
- [ ] Cookie-баннер показывается при первом визите, запоминает согласие
- [ ] Flow регистрации и заказа анализов работает как раньше
