# Sprint 1 — Подготовка к тендеру (сайт медицинской клиники)

## Цель спринта

Довести проект до состояния, пригодного для демонстрации на тендере **корпоративного сайта медицинской клиники**, не ломая и не удаляя существующий функционал **каталога анализов, корзины и оформления заказов**.

## Ключевое ограничение

> **Сохранить всю логику с анализами и покупками.**

Это означает:

- Маршруты `/catalog/*`, `/cart`, `/cart/order-conformation` остаются без изменений поведения
- Redux-слайсы `catalog`, `product`, `order`, корзина в `header-reducer` — не рефакторить ради «записи на приём»
- API-вызовы в `api_requests.js` для cart/orders/catalog — не удалять и не заменять
- Flow «добавить анализ → корзина → выбор места сдачи → оформление → заказ в ЛК» должен работать после всех доработок спринта
- Новые разделы клиники (врачи, запись, услуги) — **дополнение**, а не замена модуля диагностики

## Текущее vs целевое

| Сейчас | После спринта |
|--------|---------------|
| Портал заказа анализов | Сайт клиники **+** модуль заказа анализов |
| ~60% ссылок → заглушка InWork | Все пункты меню ведут на реальные страницы |
| Placeholder-контент | Demo-данные demo-клиники |
| Нет юридических страниц | Политика ПДн, лицензии, согласия |
| Нет разделов врачи/запись | Базовые разделы для тендерного демо |
