# 02 — Статические информационные страницы

**Приоритет:** P0  
**Оценка:** 1–2 дня  
**Зависимости:** [01-fix-navigation-and-routes](./01-fix-navigation-and-routes.md)

## Описание

Заполнить информационные разделы сайта клиники контентом вместо заглушек. Страницы могут быть статическими (JSX/JSON) или загружаться с API — на первом этапе допустим статический контент для demo.

## Ограничение

Не трогать компоненты каталога анализов, корзины и оформления заказа.

## Страницы

### О нас (`/about-us`)

- Заменить placeholder в `ContentBlock.jsx` («Текст блока будет добавлен позже»)
- История клиники, миссия, ключевые цифры (врачи, годы работы, пациенты)
- Переиспользовать блок `AboutUs` с главной, если данные приходят с API `/api/about-us/`

### Для клиентов (`/for-clients`)

- FAQ: как записаться, как подготовиться, как получить результаты анализов
- Ссылка на модуль заказа анализов (`/catalog/all-analyzes`) — **явно сохранить** как часть сервиса клиники
- Информация о домашнем визите vs приём в клинике (перекликается с существующим flow корзины)

### Партнёрам (`/for-partners`)

- B2B: корпоративное обслуживание, договоры, контакт для партнёров

### Акции (`/stocks`)

- Список акций (можно статический массив на первом этапе)
- Разкомментировать/подключить блок `Stocks` на главной (задача 09)

## Структура компонентов (предложение)

```
src/componets/InfoPages/
  AboutUsPage/
  ForClientsPage/
  ForPartnersPage/
  StocksPage/
  shared/
    InfoPageLayout.jsx    — общий layout: title + breadcrumbs + content blocks
    ContentBlock.jsx      — перенести/расширить из AboutUs/
```

## Критерии приёмки

- [ ] Каждая страница имеет заголовок, осмысленный текст (не placeholder)
- [ ] Страница «Для клиентов» содержит ссылку на каталог анализов
- [ ] Единый визуальный стиль с остальным сайтом (использовать существующие SCSS-классы)
- [ ] Страницы открываются по маршрутам из задачи 01
